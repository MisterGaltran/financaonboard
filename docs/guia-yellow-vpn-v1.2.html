01Como conectar no Yellow

A regra é simples: em casa SSH direto, fora de casa precisa VPN primeiro.

A Em casa — sem VPN

$ ssh davi@yellow-vpn.local
# OU pelo IP fixo:
$ ssh davi@192.168.1.50

B Fora de casa — precisa VPN

  1. Abrir WireGuard no notebook
  2. Ativar tunnel notebook-davi
  3. Aguardar 2-3 segundos para o túnel estabilizar
  4. Rodar SSH normalmente: ssh davi@192.168.1.50
Por que precisa VPN fora de casa: o IP 192.168.1.50 é privado da sua rede. Fora dela, ninguém consegue chegar. A VPN faz seu notebook entrar virtualmente na sua casa.

Pelo celular Android

Ativa o tunnel VPNCasa (que usa IP direto, driblando bloqueio Claro). Use apps como Termius ou JuiceSSH com IP 192.168.1.50.

Atenção Claro 5G: a Claro bloqueia o domínio davi-mendes.dedyn.io. Por isso o celular usa Endpoint por IP fixo. Se um dia o IP da Oi mudar, atualize o perfil VPN do celular.

Acesso público v1.2

Diferente da v1.0, agora o Yellow também responde a https://yellow-vpn.tailf47dec.ts.net de qualquer lugar (sem VPN), mas apenas para apps publicados via Tailscale Funnel. SSH continua restrito à LAN/VPN.

02Workflow diário com Tmux

Sessões que sobrevivem ao seu desconectar — base do trabalho remoto.

O que é tmux: programa que mantém tudo rodando no Yellow mesmo quando você fecha o SSH. Você inicia um build, fecha o notebook, vai almoçar, volta e está tudo lá.

Iniciar sessão de trabalho

$ cd /mnt/projects/financaonboard
$ tmux attach -t manutnavio || tmux new -s manutnavio

Sair sem matar (DETACH)

Ctrl + B → solta tudo → D

Você verá [detached (from session manutnavio)] e volta ao prompt normal. Tudo continua rodando!

Atalhos essenciais

03Git: sincronizando entre PC e Yellow

Pull antes de começar, push antes de fechar — sempre.

Erro mais comum: esquecer de git push antes de sair de casa. Você abre o notebook em viagem, faz git pull e não tem nada novo. Aí tem que voltar/conectar VPN só pra pushar.

Antes de COMEÇAR a programar

$ cd /mnt/projects/financaonboard   # ou pasta do PC
$ git pull

Antes de PARAR de programar

$ git status                       # o que mudou?
$ git add .                        # adiciona tudo
$ git commit -m "descrição clara"  # commit local
$ git push                         # envia pro GitHub

Como saber se tem coisa não pushada

$ git log origin/main..HEAD
# Se NADA aparecer = tudo no GitHub
# Se commits aparecerem = falta dar git push!

Atualizar o app financaonboard NOVO

Frontend: qualquer push pra main dispara rebuild automático no Cloudflare Pages.

Backend: precisa rodar o script de deploy no Yellow após o push:

$ bash /mnt/projects/financaonboard/infra/deploy.sh

O script faz git pull + npm ci --omit=dev + systemctl restart financaonboard-api e mostra status.

Cuidado com conflitos

  • Sempre git pull antes de começar
  • Trabalha em UM lugar de cada vez (não edita simultaneamente em PC e Yellow)
  • Se conflito acontecer: o Git marca o arquivo, você abre, escolhe a versão, depois git add + git commit

05Apps públicos com Tailscale Funnelv1.2

Como expor um backend do Yellow na internet sem abrir portas.

O que é Tailscale Funnel: tunnel reverso. Em vez do público bater na porta do Yellow, o Yellow inicia uma conexão outbound persistente com a infraestrutura do Tailscale, e o Tailscale expõe um endpoint público com TLS automático.

Por que não usamos Caddy + porta 443

A primeira tentativa foi Caddy + DNS-01 challenge no domínio davi-mendes.dedyn.io. Setup local funcionou, mas testes em check-host.net revelaram que o ISP Oi filtra inbound TCP 443 pra IPs residenciais: praticamente nenhum nó do mundo conseguia conectar. Migramos pra Funnel porque ele usa outbound — não passa pelo filtro.

Arquitetura

[internet] → tailnet edge → tailscaled (Yellow, outbound)
                                       │
                                       ├─ tailscaled.state (chaves, peers, config Funnel)
                                       └─ TLS auto pelo Tailscale Edge
                                       │
                                       ▼
                              localhost:4000 (financaonboard-api)
                              localhost:5000 (próximo app, via path)

Status atual

$ tailscale funnel status
# Funnel on:
#     - https://yellow-vpn.tailf47dec.ts.net
https://yellow-vpn.tailf47dec.ts.net (Funnel on)
|-- / proxy http://127.0.0.1:4000

$ curl -I https://yellow-vpn.tailf47dec.ts.net/health
HTTP/2 200

Adicionar um novo app público

Funnel só roteia pra uma porta como raiz. Pra múltiplos apps, use paths:

# reset config
sudo tailscale funnel reset

# expõe app1 em /api/financa, app2 em /api/outro
sudo tailscale serve --bg --https=443 --set-path=/api/financa http://localhost:4000
sudo tailscale serve --bg --https=443 --set-path=/api/outro http://localhost:5000
sudo tailscale funnel --bg 443 on

# verifica
tailscale funnel status

Operação do Tailscale

# status do daemon
sudo systemctl status tailscaled

# status do Funnel especificamente
tailscale funnel status

# logs ao vivo
journalctl -u tailscaled -f

# restart (preserva config Funnel)
sudo systemctl restart tailscaled

# desligar Funnel (mantém Tailscale rodando)
sudo tailscale funnel --https=443 off

# IP no tailnet (pra conexões internas)
tailscale ip -4

Renovação de certificados

Automática pelo Tailscale, ~30 dias antes do vencimento. Não faz nada.

Funnel é gratuito mas tem limite de uso. Tailscale Free tier permite Funnel mas com rate limits. Pra app pessoal de baixo tráfego, suficiente. Se virar mainstream, considera plano pago ou comprar um domínio próprio + Cloudflare Tunnel.

06Economizando dados móveis

Como minimizar consumo do 4G ao programar em viagem.

Verdade que dói: SSH/VS Code Remote-SSH NÃO economizam 4G — toda interação trafega pelo seu plano. O que economiza é deixar a IA e os builds rodando no Yellow.

Comparação de consumo (4G/h)

CenárioOnde IA rodaConsumo 4G
Cursor / Copilot LOCAL editando arquivos do YellowNotebook10-30 MB/h
Continue / Copilot REMOTO (instalado no Yellow)Yellow5-10 MB/h
Claude Code CLI no tmux do YellowYellow (CLI)1-2 MB/h
SSH puro + nano/vim (sem IDE)—~1 MB/h
Programar local + git push/pull pontual—poucos KB

Estratégias por situação

WI-FI GRÁTIS

VPN + VS Code Remote-SSH normal. Consome Wi-Fi grátis, não seu plano.

4G FOLGADO

VPN + VS Code Remote-SSH. Consumo é baixo (5-10 MB/h), tranquilo.

4G APERTADO

Edita LOCAL no notebook. Sincroniza via Git pontualmente. Consumo: KB.

SEM INTERNET

Edita local offline. Commit local, push quando voltar a ter internet.

Dicas para usar IA no Yellow

  • Continue / Copilot: instalar pelo VS Code conectado em SSH: yellow-vpn → "Install in SSH" (não local)
  • Claude Code CLI: instalar no Yellow, rodar dentro do tmux. Tudo passa pela fibra de casa
  • Cursor: é app standalone, não funciona como Remote-SSH. Para 4G limitado, evite
  • Ao mover IA para o Yellow: passar a chave da API via variável de ambiente lá também

07Backup, manutenção e troubleshooting

Quando algo der errado, ou só manutenção rotineira.

Backups automáticos

Roda automaticamente todo dia às 3h da manhã. Mantém os últimos 7 dias. Salva em /mnt/projects/backups/.

# Ver backups disponíveis
$ ls -lh /mnt/projects/backups/

# Rodar backup manual agora
$ ~/scripts/backup-projects.sh

# Ver agendamentos do cron
$ crontab -l
Backup local NÃO substitui Git. Para projetos com Git, o GitHub é seu backup principal. O backup local protege contra rm -rf acidental, mas não contra falha do SSD ou incêndio.

O que precisa de backup externo (fora do Yellow)

  • Arquivos .env dos projetos (API keys)
  • Token deSEC em /etc/caddy/desec.env
  • Chave SSH privada do Yellow (~/.ssh/id_ed25519)

Manutenção mensal

$ sudo apt update && sudo apt full-upgrade -y    # atualiza
$ sudo apt autoremove -y                          # limpa
$ df -h /mnt/projects                             # espaço SSD
$ sudo nvme smart-log /dev/nvme0n1 | head -10     # saúde SSD

Atualizar o Tailscale v1.2

O Tailscale tem repositório próprio do apt. Atualização padrão:

$ sudo apt update && sudo apt install -y --only-upgrade tailscale
$ tailscale version

Ou rerodar o instalador oficial: curl -fsSL https://tailscale.com/install.sh | sh

$ free -h                                         # memória RAM
$ df -h                                           # espaço em disco
$ sudo wg show                                    # status WireGuard
$ tail /home/davi/desec/update.log                # log DDNS
$ systemctl is-active financaonboard-api tailscaled   # apps públicos
$ tailscale funnel status                         # endpoint Funnel ativo

Emergências

  • Esqueci senha SSH: precisa monitor + teclado direto no Yellow
  • Yellow não liga: tirar fonte, esperar 30s, reconectar
  • SSD desconectou: abrir Yellow, conferir parafuso M.2
  • Perdi VPN celular: em casa, pivpn add + pivpn -qr

Workflow completo de uma sessão

  1. Conectar: SSH (com VPN se fora de casa) → tmux attach -t manutnavio
